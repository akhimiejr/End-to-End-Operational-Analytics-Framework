# End-to-End-Operational-Analytics-Framework
NYC Yellow Taxi Analytics Project : This project explores taxi transportation patterns across New York City using a robust data analytics dashboard built with Microsoft Fabric.
# 1.1 Overview 
The NYC Yellow Taxi fleet is a key component of New York
City’s vast transportation network, providing crucial mobility
services to millions of residents, commuters, and tourists.
These taxis operate throughout all five boroughs, offering an
extensive and real-time record of trips made across the city.
Each trip is meticulously tracked, capturing key details such
as pickup and drop-off locations, trip distances, fare
amounts, payment methods, and more.
The data generated by NYC Yellow Taxis offers valuable
insights that can be used to optimize city traffic patterns,
improve service efficiency, and inform policy decisions
related to transportation and urban mobility.
# 1.2 Aim of the Project
#### - Build an end-to-end data analytics pipeline using Microsoft Fabric.
#### - Analyze yellow taxi trip data to identify:
- Peak travel periods
- Common pickup/drop-off zones
- Revenue trends
- Trip characteristics (distance, duration, fares)
#### - Provide interactive dashboards for decision-makers.
#### - Showcase data engineering, data warehousing, and BI
reporting capabilities using a unified platform.





Using Microsoft Fabric’s unified experience, this case study walks through the full analytics lifecycle:
•Data Ingestion & Preparation
• Download public NYC Yellow Taxi trip data
• Upload raw files into Microsoft Fabric's One Lake Lakehouse
• Use Data Engineering (Spark Notebooks) for data cleaning and transformation
•Data Storage & Modeling
• Structure data into bronze (raw), silver (cleaned), and gold (analytics-ready) layers
• Create SQL models and views in Lakehouse & Data Warehouse
•Analytics & Visualization
• Write SQL queries to explore trip patterns, revenue, and usage trends
• Build Power BI dashboards that highlight KPIs, time trends, and location-based insights
